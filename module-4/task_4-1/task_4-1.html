        Why is object-oriented development so important in programming?
        
    Object-oriented development helps us organise code around things we want to model, such as a bank account. This makes the program easier to understand, reuse, expand, and maintain when the system becomes larger.

        In Task 1, we create a static object called "AccountType." Explain what this object is used for and how this type of object can help us in software development.
        
    The AccountType object works as a shared collection of fixed values such as Normal, Saving, and Pension. This helps us avoid spelling mistakes, makes the code easier to read, and gives us one place to change the values if the program grows later.

        Discuss why we use encapsulation in classes. Why do we want to "hide" code within our class? Explain how we can create "private" and "public" members in a constructor function.
        
    We use encapsulation to protect important data and rules inside the class, so other code cannot change everything directly in unsafe ways. In a constructor function, public members are normally added with this, while private members can be created with local variables inside the function so they are only available through public methods.

        Explain what happens when we use the "new" command in JavaScript.
        
    When we use new, JavaScript creates a new object, connects it to the class or constructor prototype, sets this to that new object, runs the constructor code, and returns the finished instance.
