  2. How do parentheses affect the arithmetic outcome of an expression?

    3. Parentheses change the order of evaluation by forcing what's inside to be calculated first. This can change the result compared to normal operator precedence (multiplication/division before addition/subtraction).

  3. How do we limit floating-point numbers to a given number of decimals?

    4. Use number.toFixed(n) to format a number with n decimals (note: it returns a string). Alternatively, use rounding math like Math.round(num * 10**n) / 10**n.

  4. How can we compare two floating-point values to see if they are equal (is 12.199999 the same as 12.2)?

    5. Avoid direct equality for floats due to precision issues. Compare with a tolerance: Math.abs(a - b) < epsilon (e.g. Number.EPSILON or a small value like 1e-9), or compare rounded values (e.g. both to 2 decimals).

  5. What do comparison operators return, and how can we check if they are FALSE or TRUE?

    6. Comparison operators return boolean values: true or false. You can store the result in a variable (e.g. const isGreater = a > b) and check it with if (isGreater) or compare strictly with === true/false when needed.

  6. How does the Math.random() function work, and what can it be used for?

    7. Math.random() returns a pseudo-random number in the range [0, 1). To get a number in a range, scale and shift: Math.floor(Math.random() * (max - min + 1)) + min. It's used for dice rolls, random IDs, random angles, sampling, etc.]

  7. What is modulus (%), and when do we need this operator in mathematical operations?

    8. The modulus operator gives the remainder after division. It's useful for checking even/odd (n % 2), wrapping values, and converting units like days to weeks and days (days / 7 and days % 7).
