    Yes. You can run one for-loop from 1 to 10 and build two output strings at the same time: one string appends i (1..10) and the other appends (11 - i) (10..1).   2. In part 2, can we create what is known as an "infinite" loop within the condition of the "while" loop? Describe what would or would not be necessary to achieve this. Provide a thorough explanation of your answer.

    Yes. A while-loop becomes "infinite" when its condition never becomes false (for example: while (true)) and there is no break, return, or error that stops it. It can also happen by mistake if the condition depends on a variable that is never updated inside the loop (e.g., the guessed number is never changed). To avoid infinite loops, you must update the state that affects the condition (or include a safe exit like a maximum attempts limit or a break when a goal is reached).

  4. Explain the relationship between rows and columns in nested "for" loops. Can we, for instance, change the order, considering rows to be innermost or columns to be innermost, or does this not matter?

    In nested loops, the outer loop usually represents rows and the inner loop represents columns because the inner loop completes all columns for one row before moving to the next row. You can swap them, but it changes how the output is grouped: you would effectively iterate "column-first" instead of "row-first" unless you also change where you print line breaks. So the order does matter for the structure/formatting of the output, even though both versions still visit all row/column combinations.
