        What is the difference between invoking and creating a function?
        
    Creating a function means defining it with a name, optional parameters, and a code block. Invoking a function means calling it so the code inside the function actually runs.

        Explain the syntax regarding semicolons and code blocks.
        
    Semicolons are normally used to end statements such as variable assignments or function expressions. Code blocks use curly braces to group statements together inside functions, loops, and conditional statements.

  3. 
        What happens when we try to store values from a function when it doesn't return any?
        
    If a function does not use return, JavaScript returns undefined. That means the variable we store the result in will contain undefined, even if the function printed something.

        How do we separate parameters from each other when creating a function?
        
    Parameters are separated by commas inside the parentheses, for example function example(a, b, c).

  5. 
        Explain how parameter names are related during the invocation and creation of a function.
        
    Arguments are matched to parameters by their position, not by their names. The parameter names only exist inside the function, so the caller can pass values from variables with completely different names.

  6. 
        Explain why a parameter is undefined when a function call is missing a parameter.
        
    When an argument is left out in a function call, JavaScript still creates the parameter, but gives it the value undefined. The function must then handle that value with checks or default parameters.

  7. 
        Explain what happens when we use recursive functions.
        
    A recursive function calls itself. Each call gets its own local variables and runs until a base case stops the recursion; without a base case, the program will eventually hit a stack overflow.
